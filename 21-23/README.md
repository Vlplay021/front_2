# Контрольная работа 4

В ходе выполнения контрольной работы были реализованы практические задания №21–23 (21-24) по дисциплине *«Фронтенд и бэкенд разработка»*.

---

## Практическая работа 21

- реализовано REST API на Node.js (Express) для работы с пользователями и товарами
- добавлена аутентификация с использованием JWT (access + refresh токены)
- реализована ролевая модель доступа (RBAC)
- добавлено кэширование данных с помощью Redis:
  - `/api/users`, `/api/users/:id` — кэш 1 минута
  - `/api/products`, `/api/products/:id` — кэш 10 минут
- реализован middleware для кэширования (`cacheMiddleware`)
- реализована очистка кэша при изменении данных пользователей

---

## Практическая работа 22

- реализован backend-сервис на Express, отвечающий на GET `/`
- настроена балансировка нагрузки с использованием Nginx
- реализовано распределение запросов между backend-сервисами (round-robin)
- добавлены параметры отказоустойчивости:
  - `max_fails`
  - `fail_timeout`

---

## Практическая работа 23

- реализована контейнеризация приложения с помощью Docker
- создан `Dockerfile` для backend-сервиса
- настроен `docker-compose.yml` для запуска нескольких сервисов:
  - backend (Node.js)
  - Redis
  - Nginx (балансировщик нагрузки)
- все сервисы объединены в одну сеть Docker
- реализован запуск всего проекта одной командой:

```bash
docker compose up --build 
```
