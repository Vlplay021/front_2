# PWA «Заметки» — Офлайн + WebSocket + Push-уведомления

**Прогрессивное веб-приложение (PWA)** для создания заметок и напоминаний с полной офлайн-поддержкой, синхронизацией в реальном времени и push-уведомлениями, работающими даже при закрытом браузере.

## Основной функционал

| Возможность | Реализация |
|-------------|-------------|
| Офлайн-доступ | Service Worker кэширует статику и динамические страницы |
| Установка на устройство | Web App Manifest + иконки для всех разрешений |
| Синхронизация в реальном времени | WebSocket (Socket.IO) — заметки появляются у всех открытых клиентов |
| Push-напоминания | Отложенные уведомления через Web Push API |
| Отложить напоминание | Кнопка в уведомлении переносит событие на +5 минут |
| Постоянное хранение | Заметки хранятся в `localStorage`, синхронизация по `id` без дублей |

## Потоки данных

### 1. Обычная заметка
1. Пользователь вводит текст → `addNote()`
2. Сохраняется в `localStorage`
3. Отправляется через WebSocket (`socket.emit('newTask')`)
4. Сервер рассылает `taskAdded` всем клиентам
5. Другие клиенты получают → проверяют `id` → сохраняют в свой `localStorage` → обновляют интерфейс

### 2. Заметка с напоминанием
1. Пользователь выбирает дату/время → `addReminderNote()`
2. Сохраняется в `localStorage`
3. WebSocket отправляет два события:
   - `newTask` — для синхронизации самой заметки
   - `newReminder` — для планирования push-уведомления на сервере
4. Сервер запускает `setTimeout()` на разницу `reminderTime - Date.now()`
5. По истечении времени — рассылает push всем подписанным клиентам

### 3. Откладывание напоминания
1. Пользователь нажимает "Отложить на 5 минут" в уведомлении
2. Service Worker отправляет `POST /snooze?reminderId=...`
3. Сервер очищает старый таймер, запускает новый через 5 минут
4. Через 5 минут приходит новое push-уведомление

## API сервера (Express)

| Метод | Эндпоинт | Назначение |
|-------|----------|-------------|
| `POST` | `/subscribe` | Сохранить push-подписку (тело — объект подписки) |
| `POST` | `/unsubscribe` | Удалить подписку по `endpoint` |
| `POST` | `/snooze?reminderId=...` | Отложить напоминание на 5 минут |
| `*` | `/*` | Раздача статики (`express.static`) |

### WebSocket-события (Socket.IO)

| Событие (клиент → сервер) | Данные | Действие сервера |
|--------------------------|--------|------------------|
| `newTask` | `{ id, text, datetime, reminder }` | Рассылает `taskAdded` всем клиентам |
| `newReminder` | `{ id, text, reminderTime }` | Запускает таймер push-уведомления |

| Событие (сервер → клиент) | Данные | Действие клиента |
|---------------------------|--------|------------------|
| `taskAdded` | `{ id, text, datetime, reminder }` | Сохраняет в `localStorage` и обновляет список |

## Технологии

- **Frontend**: Vanilla JS, Chota CSS (микро-фреймворк)
- **PWA**: Service Worker, Web App Manifest
- **Realtime**: Socket.IO (клиент 4.5.4, сервер 4.x)
- **Push**: Web Push API + `web-push` библиотека
- **Backend**: Express.js, HTTP, body-parser, cors
- **Хранение**: `localStorage` (клиент), в памяти сервера (подписки и таймеры)

## Запуск проекта (пошагово)

### 1. Установка зависимостей
```bash
npm install express socket.io web-push body-parser cors
```
### 2. Генерация VAPID-ключей (если нужны новые)
```
npx web-push generate-vapid-keys
```
Скопируйте публичный и приватный ключи в server.js и публичный ключ в app.js (VAPID_PUBLIC_KEY).

### 3. Запуск сервера
```
node server.js
```
Сервер запустится на `http://localhost:3001`

## Открытие приложения

Перейдите в браузере: [http://localhost:3001](http://localhost:3001)

## Проверка ключевых сценариев

- **Офлайн:** отключите сеть → страницы загружаются из кэша, можно добавлять заметки (они сохранятся локально)
- **WebSocket:** откройте два окна → добавьте заметку в одном → она появится во втором
- **Push:** нажмите "Включить уведомления", разрешите доступ → создайте напоминание в будущем → дождитесь push

## Важные замечания

### HTTPS / localhost
Push-уведомления работают только через HTTPS (или localhost).

При продакшене используйте реальный HTTPS-сервер (например, с Let's Encrypt).

Для локальной разработки localhost — исключение, браузер разрешает.

### Хранилище подписок и таймеров
В текущей версии подписки хранятся в массиве в памяти — при перезапуске сервера они теряются.

Для продакшена нужно использовать Redis или БД (PostgreSQL, MongoDB).

### Таймеры напоминаний
Используется `setTimeout()` — при перезапуске сервера все запланированные напоминания сбрасываются.

Для надёжной работы нужна очередь задач (Bull, Agenda) с персистентностью.

### Service Worker и кэш
При изменении `sw.js` увеличьте `CACHE_NAME` (например, `notes-cache-v4`).

Браузер автоматически установит новый SW после перезагрузки страницы.

## Возможные проблемы и решения

| Проблема | Вероятная причина | Решение |
|----------|------------------|---------|
| Push-уведомления не приходят | Неправильные VAPID-ключи или HTTP вместо HTTPS | Проверьте ключи, откройте через localhost |
| Дублирование заметок | WebSocket успевает прислать копию раньше, чем сохранится локальная | В коде есть проверка `!notes.some(n => n.id === task.id)` |
| Не работает "Отложить" | Service Worker не может выполнить fetch('/snooze') | Убедитесь, что сервер доступен и CORS настроен |
| После установки PWA пропадают иконки | Пути к иконкам в manifest.json не совпадают | Проверьте регистр имён файлов и папку /icons/ |
| WebSocket не подключается | Сервер не запущен или порт занят | Запустите `node server.js`, проверьте порт 3001 |
